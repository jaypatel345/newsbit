# AI Integration

This document describes how AI is used in **Newsbit AI – Version 1**.

The purpose of AI in Version 1 is to generate concise, factual summaries of the latest news articles.

---

# AI Provider

**Provider**

Groq API

**Purpose**

Generate high-quality summaries for news articles.

---

# AI Workflow

```text
News API
    │
    ▼
Fetch Latest Articles
    │
    ▼
Store Articles in Database
    │
    ▼
Find Articles Without Summary
    │
    ▼
Send Article Content to Groq API
    │
    ▼
Generate Summary
    │
    ▼
Store Summary in Database
    │
    ▼
Return Summary to Frontend
```

The frontend never communicates directly with the AI provider.

All AI requests are handled by the FastAPI backend.

---

# AI Model

Version 1 uses a Groq-hosted Large Language Model (LLM) for summarization.

The specific model may change over time without affecting the rest of the application.

---

# Prompt Objective

The AI should generate a concise summary that allows users to understand the key points of a news article in less than one minute.

The summary should:

- Focus only on the important information.
- Be clear and easy to understand.
- Remain factual and neutral.
- Avoid opinions or speculation.
- Ignore advertisements or unrelated content.

---

# AI Input

The backend sends the following information to the model:

- Article title
- Article description
- Article content (when available)

---

# AI Output

The model returns a plain-text summary.

The generated summary is stored in the `Article` table to avoid generating it again for future requests.

---

# Error Handling

The AI service should gracefully handle the following situations:

- Empty article content
- AI request timeout
- API rate limits
- Invalid AI response
- Network failures

If summary generation fails, the article remains available and can be summarized later.

---

# Design Decisions

## Backend-Controlled AI

All AI interactions occur on the backend.

This keeps API keys secure and centralizes prompt management.

---

## Cached Summaries

Summaries are generated once and stored in the database.

This reduces response time, lowers API usage, and ensures consistent results for all users.

---

## Prompt Management

AI prompts are stored separately inside the backend (`app/prompts`) rather than being hardcoded into business logic.

This makes prompts easier to update, test, and improve without modifying application code.

---

# Future Enhancements

Future versions may expand AI capabilities with features such as:

- Personalized news recommendations
- Multi-language summaries
- Adjustable summary length
- Follow-up AI chat about articles
- Article categorization
- AI-powered search
- Personalized news feeds
